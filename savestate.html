SAVE STATE

UNREAL ENGINE 5 // C++ // PHYSICS PUZZLE

MISSION BRIEFING: O Save State é um jogo de puzzles na primeira pessoa onde o jogador tem de reparar máquinas complexas. O twist mecânico é a validação física precisa: as peças não "encaixam" sozinhas por magia, elas têm de estar alinhadas e aparafusadas corretamente.

/// TECHNICAL CHALLENGE: SOCKET SYSTEM

PROBLEM DETECTED

O sistema de 'Attachment' nativo do Unreal era demasiado simples. As peças atravessavam-se umas às outras (clipping) e não havia sensação de peso ou resistência ao encaixar.


SOLUTION DEPLOYED

Desenvolvi um sistema de Socket Validation em C++. Criei uma classe `AMechanicPart` que verifica a colisão antes de permitir o encaixe. Usei `SweepSingleByChannel` para prever se a peça vai bater em algo antes de chegar ao destino, criando uma física "snappy" mas realista.

/// ARCHITECTURE & SCALABILITY

Para garantir que podíamos criar 50 puzzles diferentes sem reescrever código, utilizei Data Assets. Cada máquina é definida por um Data Asset que lista quais as peças necessárias e as suas posições.

Isto permitiu aos Game Designers criar níveis novos sem tocar numa linha de C++.

/// MISSION DEBRIEF

Este projeto fortaleceu o meu conhecimento em C++ puro e matemática vetorial (Dot Products foram essenciais para calcular o alinhamento das peças). O resultado final é um sistema robusto que pode ser reutilizado em qualquer jogo de simulação mecânica.
